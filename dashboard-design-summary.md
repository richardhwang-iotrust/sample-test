# Dashboard Design Summary

## 1. 문서 목적

이 문서는 현재 `recruit-dashboard`의 대시보드 UI 디자인을 빠르게 파악하고, 이후 화면 개편이나 신규 페이지 추가 시 일관성을 유지하기 위한 기준 문서다.  
정리 대상은 주로 아래 구현을 기준으로 한다.

- `src/app/dashboard/page.tsx`
- `src/app/dashboard/positions/page.tsx`
- `src/app/dashboard/team/page.tsx`
- `src/app/dashboard/DashboardLayoutClient.tsx`
- `src/components/Sidebar.tsx`
- `src/app/globals.css`

## 2. 전체 디자인 방향

현재 대시보드 디자인은 채용 운영 도구에 맞춘 **밝은 업무형 UI**에 가깝다.

- 기본 배경은 `bg-gray-50` 중심의 밝은 톤
- 주요 정보는 `bg-white + border-gray-200 + rounded-xl` 카드로 구획
- 강조색은 `indigo`를 중심으로, 상태별 의미색을 함께 사용
- 텍스트 위계는 `gray-900 / gray-700 / gray-500 / gray-400` 단계로 정리
- 차트는 Metabase 스타일에 가까운 단순하고 해석이 쉬운 구성
- 복잡한 장식보다 “빠른 현황 파악”과 “운영 액션 연결”을 우선

핵심 인상은 화려함보다 **실무형 대시보드**, **낮은 학습비용**, **정보 밀도 대비 높은 가독성**이다.

## 3. 레이아웃 구조

### 3.1 공통 레이아웃

대시보드는 좌측 사이드바 + 우측 메인 콘텐츠 구조다.

- 좌측: 포지션/콘텐츠/소싱으로 나뉜 탐색 네비게이션
- 우측: 페이지별 KPI, 차트, 리스트, 관리 액션
- 전체 높이: `min-h-screen`
- 메인 배경: `bg-gray-50`
- 콘텐츠 영역: `p-4 md:p-6`, `space-y-*`로 세로 리듬 유지

### 3.2 모바일 대응

모바일에서는 사이드바가 오프캔버스 패널로 동작한다.

- 햄버거 버튼으로 열기
- `fixed inset-0` 오버레이 사용
- 상단에 별도 모바일 헤더 제공
- 데스크톱에서는 고정형 사이드바, 모바일에서는 슬라이드형 내비게이션

이 패턴 덕분에 정보 구조는 유지하면서도 작은 화면에서 작업 흐름이 끊기지 않는다.

## 4. 주요 UI 패턴

### 4.1 카드 패턴

가장 많이 반복되는 기본 패턴은 아래와 같다.

- 배경: 흰색
- 모서리: `rounded-xl`
- 테두리: `border border-gray-200`
- 내부 여백: `p-4` 또는 `p-5`

대표 컴포넌트 성격:

- KPI 카드: 수치 중심, 아이콘 + 라벨 + 값 + 보조 설명
- 차트 카드: 제목 + 서브카피 + 시각화
- 그룹 카드: 팀/권한/멤버 목록 등 관리형 정보 블록

### 4.2 타이포그래피

- 페이지 제목: `text-xl font-bold text-gray-900`
- 섹션/카드 제목: `text-sm font-semibold text-gray-800`
- 본문/설명: `text-sm text-gray-500`
- 보조 설명/라벨: `text-xs text-gray-400~500`
- KPI 수치: `text-2xl font-bold`

전체적으로 텍스트 크기 차이를 크게 벌리기보다, 작은 단위의 위계를 촘촘히 쌓는 방식이다.

### 4.3 인터랙션

- hover는 강한 변형보다 배경색 변화 중심
- 선택 상태는 주로 `bg-indigo-50 text-indigo-700`
- 클릭 가능한 요소는 `rounded-lg`로 통일감 확보
- 관리성 인터랙션은 드래그 앤 드롭, 접힘/펼침, 상태 토글 중심

## 5. 컬러 시스템

### 5.1 공통 베이스

- 페이지 배경: `gray-50`
- 카드 배경: `white`
- 기본 보더: `gray-200`
- 주요 텍스트: `gray-900`, `gray-800`
- 보조 텍스트: `gray-500`, `gray-400`
- 핵심 포인트 컬러: `indigo`

### 5.2 KPI 카드 컬러

홈 대시보드 KPI는 아래 의미색을 사용한다.

- `indigo`: 전체 규모, 기본 핵심 지표
- `green` / `emerald`: 합격, 긍정 전환 등 좋은 성과
- `amber`: 중간 경고, 목표 미달 가능성
- `rose`: 낮은 성과, 주의 필요
- `violet`: 소싱/보조 영역

아이콘 배경과 숫자 컬러를 같은 계열로 맞춰, 작은 카드에서도 의미 전달이 빠르다.

### 5.3 전형 단계 컬러

전형 단계는 고정된 의미색을 가진다.

- 서류접수: `#14b8a6`
- 서류검토: `#6366f1`
- 실기: `#a855f7`
- 면접: `#eab308`
- 처우협의: `#f97316`
- 최종합격: `#22c55e`
- 불합격: `#9ca3af`

퍼널, 단계별 차트, 상태 표시에서 동일한 색 체계를 재사용하는 것이 중요하다.

### 5.4 소싱 상태 컬러

소싱 대시보드는 별도 상태 팔레트를 사용한다.

- 메시지발송: indigo 계열
- 답변대기: amber 계열
- 긍정반응: green 계열
- 부정반응: red 계열
- 인터뷰진행: violet 계열
- 처우합의: orange 계열
- 최종합격: green 계열
- 재연락: cyan 계열
- 거절: gray 계열

즉, 전체 시스템은 “인바운드 전형”과 “소싱 상태”를 각각 독립된 상태 집합으로 다룬다.

## 6. 페이지별 디자인 특징

### 6.1 홈 대시보드

홈 대시보드는 여러 데이터를 한눈에 보는 **임원/리더용 요약 화면** 성격이 강하다.

- 상단 헤더 + 빠른 이동 버튼
- 5개 KPI 카드
- 2열 차트 그리드 반복
- 퍼널, 추이, 포지션 분포, 소싱 분포, 채널 분석, 목표 달성 현황 제공

특징:

- 대부분의 차트가 “현재 상황을 설명”하는 용도
- 카드 간 여백이 충분해 시선 이동이 편함
- 요약성과 해석 용이성을 우선해 차트 종류를 제한적으로 사용

### 6.2 인바운드 전체 현황

`positions/page.tsx`는 홈보다 더 운영자 친화적이다.

- 필터, 토글, 편집, Slack 알림 같은 액션이 추가
- KPI와 차트를 유지하면서도 관리 흐름이 강화됨
- 변경 이력, 목표값 수정, 포지션별 제어가 함께 배치됨

즉, 홈이 “요약”이라면 인바운드 전체 현황은 “운영 콘솔”에 가깝다.

### 6.3 참여자 현황

`team/page.tsx`는 카드 기반 정보 관리 화면이다.

- 역할별 그룹 카드 배치
- 아바타, 멤버 수, 권한 설명, 배정 포지션 노출
- 제거/복구 같은 관리 액션 포함

같은 디자인 시스템을 쓰되, 차트보다 리스트와 관계 정보 표현에 집중한다.

## 7. 사이드바 디자인 정리

사이드바는 단순 메뉴가 아니라 정보 구조를 설명하는 장치다.

- 상단 브랜드 영역 + 현재 사용자 역할 배지
- 섹션 라벨로 정보 군집 분리
- 폴더/서브폴더 구조로 정보 밀도 제어
- 진행중/종료 구분에 상태 점 사용
- 현재 선택 메뉴는 indigo 배경으로 명확히 강조

좋은 점:

- 기능 수가 늘어나도 확장성이 좋음
- 권한별 메뉴 제어와 구조적 탐색이 자연스럽게 연결됨
- 드래그 앤 드롭까지 포함해 운영 도구다운 성격이 살아 있음

주의할 점:

- 메뉴 트리가 더 깊어지면 모바일 탐색 부담이 커질 수 있음
- 현재는 텍스트 밀도가 높아, 섹션 수가 더 늘면 시각적 피로가 생길 수 있음

## 8. 차트 디자인 원칙

현재 차트 구성에서 반복되는 원칙은 아래와 같다.

- 격자선은 옅게: `#f3f4f6`
- 축 라벨은 작고 연하게: `text-xs` 이하, `gray-400~500`
- 툴팁은 흰색 카드형 + 얇은 보더
- 데이터 라벨은 필요한 곳에 항상 노출
- 범례는 차트 바깥에 단순 점/선 형태로 제공
- 색 수는 최소화하고 의미 기반으로 유지

즉, 차트는 “정교한 분석용”보다 **바로 읽히는 운영형 데이터 시각화**를 지향한다.

## 9. 권한에 따른 디자인 차이

이 프로젝트는 UI 자체가 권한 모델과 강하게 연결되어 있다.

- 조회 불가 데이터는 숨기거나 `dim` 처리
- 일부 페이지는 접근 차단 화면으로 대체
- 특정 역할만 관리 버튼, 소싱 화면, 팀 관리 기능 노출

디자인 측면에서 중요한 점은 “권한 없음”을 단순 에러가 아니라 명확한 상태 화면으로 표현한다는 것이다.

## 10. 현재 디자인의 강점

- 실무 대시보드로서 가독성이 높다
- 카드/차트 패턴이 일관적이라 신규 화면 확장에 유리하다
- 상태 색상 체계가 분명하다
- 모바일 대응이 현실적이다
- 권한 기반 UX가 구조적으로 잘 연결되어 있다
- 요약 화면과 운영 화면의 성격 차이가 비교적 명확하다

## 11. 개선 여지

### 11.1 토큰화 부족

현재 컬러와 카드 스타일이 Tailwind 유틸리티와 상수로 분산되어 있어, 디자인 시스템 문서화나 테마 확장이 어렵다.

추천:

- 공통 카드 스타일을 재사용 컴포넌트로 분리
- 상태 컬러를 별도 토큰 파일로 통합
- KPI/차트/상태 배지를 공통 스타일 규약으로 묶기

### 11.2 페이지 간 스타일 기준의 암묵성

패턴은 반복되지만 “언제 어떤 카드 타입을 써야 하는지”가 코드에만 존재한다.

추천:

- `SummaryCard`, `MetricCard`, `ChartCard`, `AdminCard` 등 명명 규칙 정리
- 페이지 템플릿 수준의 레이아웃 가이드 정의

### 11.3 대시보드와 퍼블릭 사이트 테마 분리 필요

`globals.css`에는 퍼블릭 사이트용 다크 테마 변수와 대시보드용 밝은 스타일이 함께 존재한다.

추천:

- 사이트/대시보드 스타일 영역을 더 명확히 분리
- 추후 스코프 분리 또는 별도 theme layer 고려

## 12. 유지보수 시 권장 원칙

앞으로 대시보드 화면을 추가하거나 수정할 때는 아래 원칙을 권장한다.

1. 새 페이지도 기본적으로 `gray-50` 배경 + 흰 카드 패턴을 유지한다.
2. KPI는 4~6개 이내로 제한하고, 카드 한 장에는 한 가지 핵심 메시지만 담는다.
3. 상태 색상은 기존 단계/소싱 팔레트를 우선 재사용한다.
4. 차트는 장식보다 해석 속도를 우선한다.
5. 권한 차이는 숨김, 비활성, 접근 차단 화면 중 하나로 일관되게 처리한다.
6. 모바일에서는 “가로 비교”보다 “세로 흐름”이 유지되도록 카드 순서를 설계한다.

## 13. 한 줄 요약

현재 대시보드 디자인은 **밝은 B2B 운영형 UI**, **강한 카드 구조**, **명확한 상태 색상**, **차트 중심 요약 경험**을 기반으로 하며, 향후에는 이를 공통 토큰과 재사용 컴포넌트 단위로 더 명확하게 시스템화하는 것이 가장 큰 개선 포인트다.
