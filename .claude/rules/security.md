# 보안 규칙

> 이 프로젝트 사용자는 비개발자입니다. 보안 개념을 어렵게 설명하지 말고, **무엇을 하면 안 되는지 / 대신 뭘 하면 되는지**를 그때그때 쉬운 말로 안내한다.

## Claude가 항상 깔고 가야 할 태도

1. **비밀값이 대화창·코드·커밋·스크린샷에 노출되는 걸 먼저 막는다**
2. 사용자가 실수로 잘못할 수 있는 지점을 **미리 설명**한다
   - 예) "이 값은 `.env`에만 적어두시고, 다른 파일이나 대화창에 붙여넣지 마세요"
3. **인증 방식은 권장 계층대로 제안**한다 (아래 2절 참고). Classic PAT은 절대 제안하지 않는다.

---

## 1. 비밀값 취급 — 절대 기준

"비밀값" = API 키, 액세스 토큰, PAT, 비밀번호, DB 접속 정보, 암호화 키, webhook secret 등

### 🚫 절대 하지 않는 것
- 비밀값을 **코드 파일에 직접 써넣기** (예: `const API_KEY = "sk-..."`)
- 비밀값이 담긴 파일(`.env`, `credentials.json` 등)을 **커밋·푸시**
- 비밀값을 **대화창에 복사해서 올려받기** ("토큰 뭐야?" 라고 묻지 않는다)
- 비밀값을 **평문으로 출력하는 명령** 실행 (`cat .env`, `echo $TOKEN` 등)
- 비밀값을 **로그, 터미널 히스토리, Claude 대화 기록**에 남기기

### ✅ 대신 이렇게 한다
- `.env` 파일 사용, **`.gitignore`에 `.env` 포함 여부 먼저 확인**
- 코드에서는 환경변수로 참조 (예: `process.env.API_KEY`)
- **비밀값을 써야 할 땐 사용자가 직접 `.env`에 붙여넣도록 안내**한다
  - 예) "API 키는 직접 `.env` 파일을 열어 `OPENAI_KEY=여기에붙여넣기` 형식으로 추가해 주세요. 대화창에는 복사해서 보내지 마세요."

---

## 2. 외부 서비스 연동 제안 시 인증 방식 선택

새 서비스(GitHub, Notion, Slack, OpenAI, AWS 등)를 연결하려 할 때, Claude는 **먼저 용도에 맞는 인증 방식을 고른다.** 무조건 "토큰 발급해서 주세요"라고 하지 않는다.

| 용도 | 권장 방식 | 피해야 할 것 |
|---|---|---|
| 로컬에서 일상 git 작업 (clone/push/pull) | **SSH Key** (ed25519 + passphrase) | PAT을 `.git/config`에 평문 박기 |
| `gh` CLI / GitHub 명령 | **`gh auth login`** (OAuth) | 수동 PAT 발급 |
| 커스텀 스크립트의 GitHub API 호출 | **Fine-grained PAT** (만료일 90일, scope 최소) | Classic PAT |
| 서버·CI·봇 자동화 | **GitHub App** (Installation Token, 1시간 자동 만료) | 공유 PAT |
| GitHub Actions 내부 | **`GITHUB_TOKEN`** (자동 주입) | 별도 PAT |
| 단일 레포 배포 서버 | **Deploy Key** (레포 한정, 가능하면 read-only) | 광범위 PAT |

### Classic PAT 관련
- **신규 발급을 제안하지 않는다.** 이미 있더라도 fine-grained로 마이그레이션 권장.
- 사용자가 "PAT 만들어야 해?"라고 물으면 먼저 **fine-grained로도 되는지** 확인.

## 3. 커밋 전 자동 점검 (자동화)

`git commit` 명령은 `.claude/hooks/guard.sh`에서 다음을 자동으로 검사한다:

1. **staged 파일에 실제 `.env` 계열 파일**이 포함됐는지 (`.env`, `.env.local`, `.env.production` 등 — `.env.example` 제외)
   - 포함 → 커밋 차단, `.gitignore` 등록 + `git rm --cached <파일>` 안내
2. **staged diff에 토큰 패턴**이 새로 추가됐는지
   - 차단 패턴: `ghp_`, `github_pat_`, `gho_`, `sk-ant-`, `sk_live_`, `AKIA...` (AWS) 등
   - 포함 → 커밋 차단, 해당 토큰을 발급처에서 즉시 revoke + 코드에서 제거 안내

사용자에게 설명할 때:
> "커밋 전에 비밀값이 섞여있는지 자동으로 확인해요. 걸리면 일단 멈추고 알려드려요."

## 4. 사용자에게 비밀값 입력을 요청할 때

`.env` 채우기, API 키 설정 등 비밀값이 필요한 단계:

### 하지 말 것
- "토큰 여기 붙여주세요" 같이 **대화창에 붙여넣게 하지 말 것** (대화 기록에 평문으로 영구 보관됨)
- "제가 `.env`에 써둘게요" 하면서 값 달라고 하지 말 것

### 해야 할 것
- **값의 출처(1Password·회사 비밀값 관리 도구)만 안내**하고, 직접 터미널/에디터에서 `.env`에 붙여넣도록 유도
- 입력 완료 여부만 응답 받기 — 예) "파일에 넣었어요" → "확인하고 이어서 진행할게요"

예시 멘트:
> "API 키는 1Password의 'OpenAI-API' 항목에 있는 값을 복사해서, 에디터에서 `.env` 파일을 열고 `OPENAI_API_KEY=복사한값` 형식으로 직접 붙여넣어 주세요. **대화창에는 보내지 마세요.** 다 되면 '넣었어'라고만 말씀해 주세요."

---

## 5. 스크린샷·로그 공유 시 마스킹 체크

사용자가 이미지·터미널 로그를 공유했을 때 Claude는 **먼저 비밀값이 보이는지 확인**한다:

- 토큰 패턴이 보이면(`ghp_...`, `gho_...`, `sk-ant-...`, `sk_live_...`, `AKIA...` 등) **즉시 알림**
- 멘트 예시:
  > "공유해주신 화면/로그에 토큰 값이 그대로 보여요. 이 값은 **지금 바로 해당 서비스에서 revoke**(사용 중단)하고 새로 발급받는 게 안전해요. 한 번 이렇게 노출되면 스크린샷·캐시·대화 로그 어디에 남아있을지 모르거든요."
- Claude는 이후 응답에서 해당 토큰 값을 **다시 언급하거나 복사하지 않는다** (분석에 필요한 경우 마스킹: `ghp_...7f3a` 형태)
