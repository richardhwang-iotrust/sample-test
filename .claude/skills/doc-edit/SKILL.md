# Markdown 문서 편집

기획/스펙 Markdown 문서를 구조 유지하면서 편집한다.

## 진행 방식

1. 대상 파일을 읽고 전체 구조(헤딩 트리)를 먼저 파악한다
2. 사용자에게 어느 섹션을, 어떻게(추가/수정/삭제) 바꿀지 확인한다
3. 변경이 다른 섹션·TOC·내부 링크에 영향을 주는지 체크한다

## 편집 원칙

- 기존 헤딩 레벨(h1/h2/h3) 구조를 깨지 않는다
- 섹션 제목이 바뀌면 TOC·내부 링크도 함께 동기화한다
- 어조(존댓말 / 기술 문서 톤)를 기존 문서와 맞춘다
- 빈 줄·`---` 구분선 등 기존 파일의 포맷 패턴을 따라간다
- 한 번에 여러 섹션을 바꿀 땐 Edit 을 여러 번 호출해 변경 흔적을 명확히 남긴다

## 편집 후

- 무엇이 어디서 어떻게 바뀌었는지 섹션 단위로 요약한다
- 이 변경으로 수정이 필요한 다른 문서(상호 링크·참조)가 있으면 함께 알린다
