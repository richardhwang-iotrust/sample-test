세션 시작 준비를 순서대로 진행한다.

1. `git config user.email` 실행 → `@` 앞 이름을 브랜치 prefix로 파악하고 사용자에게 알려준다
2. `git branch --show-current` 실행 → 현재 브랜치 확인
   - 브랜치가 `main`이면: 새 작업 브랜치가 필요하다고 안내한다
   - 브랜치가 작업 브랜치면: 어떤 작업을 하던 브랜치인지 이름 기반으로 파악해서 말해준다
3. `git pull origin main` 실행 → 최신 상태로 업데이트
4. `git status` 실행 → 미완료(미커밋) 작업이 있으면 사용자에게 알려준다
5. 준비 완료 상태를 한 줄로 요약해서 전달한다

예시 완료 메시지:
> 준비 완료! 작업자: sumin / 현재 브랜치: sumin-fix-email / 최신 main 반영됨 / 미완료 작업 없음
